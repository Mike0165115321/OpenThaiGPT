# 03 RAG Engine (เครื่องยนต์ระบบดึงความรู้)

เอกสารฉบับนี้โฟกัสที่จุดเด่นของแอปพลิเคชัน นั่นคือการดึงสัจธรรม (Knowledge) จากภายนอกมาป้อนให้ AI ระบบ RAG ของเราสร้างขึ้นผ่าน `build_index.py` (สร้างฐานข้อมูล) และ `rag_engine.py` (ทำการค้นหา)

---

## 1. Vector Embeddings (คณิตศาสตร์ของการแปลงคำเป็นตัวเลข)

โมเดลที่รับหน้าที่เป็นช่างแปลความหมายคือ `BAAI-bge-m3` งานของมันไม่ใช่การ "จำ" คีย์เวิร์ด แต่คือการ "เข้าใจบริบท" (Semantic Search) 

เมื่อข้อมูลแต่ละท่อนจากหนังสือเดินผ่าน Embedder มันจะถูกแปลงเป็น "ลูกศรทิศทางในอวกาศความรู้" (Vector) ขนาดยาวหลายร้อยมิติ 
*ตัวอย่างเช่น คำว่า "หมา" และ "สุนัข" จะมีเวกเตอร์ชี้ไปในทิศทางอวกาศใกล้เคียงกันมาก*

---

## 2. FAISS (Facebook AI Similarity Search) Database

FAISS คือห้องสมุดที่ใช้ค้นหาหนังสือที่เร็วที่สุดในโลก การทำงานแบบ In-Memory

### สร้างฐานข้อมูล (`build_index.py`)
ระบบจะรวมหนังสือทุกหน้า โยนเข้าโมเดลเพื่อสร้างค่า Vector และยัดลงไปในกล่องที่ชื่อ `faiss_index.bin` ซึ่งถูกออกแบบด้วยโครงสร้าง `IndexFlatL2` (ค้นหาเชิงเรขาคณิตด้วยความแม่นยำ 100% แบบเรียงตัว)
*   **Proactive Optimization:** เราทำกระบวนการนี้ล่วงหน้าแบบ Offline (Batch processing) เพราะถ้ารันสดตอนผู้ใช้ถาม ระบบจะช้าเป็นสิบนาที

### กระบวนการสืบค้น (Search & Retrieval)
เส้นทางการดึงข้อมูลเมื่อมี `query` ผ่านเข้ามาใน `rag_engine.py`:
1. **Encoding:** แปลงคำถาม `query` ให้เป็น Vector บนพื้นที่อวกาศเดียวกันกับหนังสือ (รันบน CPU เพื่อประหยัด VRAM ให้ตัว LLM!)
2. **L2 Distance Calculation:** ยิงเรดาร์วัดระยะทางหาค่าวัดแบบฉากฉาก (L2 / Euclidean Distance) ใน FAISS
3. **Similarity Score Mapping:** จากสมการ `score = 1 / (1 + distance)` ยิ่ง L2 ใกล้ 0 คะแนน Score จะยิ่งพุ่งเข้าใกล้ 1

## 3. The Relevance Threshold Routing (กฎควบคุมจราจร RAG)

การมี RAG ไม่งั้นแปลว่าต้องยัด RAG ตลอดเวลา ใน `ai_core.py` มีกลไก Routing อัจฉริยะ:

```python
RELEVANCE_THRESHOLD = 0.60
is_relevant = best_score >= RELEVANCE_THRESHOLD
```

**อัลกอริทึม Routing (การแบ่งโหมดอัตโนมัติ):**
*   **ถ้าคะแนน (Score) เยอะกว่า 0.60:** เป็นไปได้สูงว่าผู้ใช้ถามคำถามที่มีสาระตรงกับในหนังสือ ระบบจะสั่งยัดก้อนบทความจากหนังสือเข้าสู่ Prompt ของ LLM และบีบบังคับ LLM ให้สวมบทบาท "คิวเรเตอร์" (Curator)
*   **ถ้าคะแนนต่ำกว่า 0.60:** (เช่น พิมพ์มาว่า "หวัดดีครับ กินข้าวหรือยัง") เวกเตอร์จะไม่แมตช์กับตำราเรียนเลย ระบบจะฉลาดพอที่จะ **"โยน RAG ทิ้ง"** และปรับใช้โหมด "แชทปกติ (Conversational)" ทันที

**ทำไมการตัดสินใจนี้ถึงสำคัญ?**
ถ้าไม่มี Routing, AI จะเกิดอาการ "RAG Hallucination" คือพยายามดันทุรังเอาหนังสือฟิสิกส์ มาตอบคำถามการทักทาย ซึ่งมักจะส่งผลให้ได้คำตอบที่ประหลาดกวนประสาท (เช่น ผู้ใช้ถาม: "นายชื่ออะไร", AI ตอบ: "อ้างอิงจากบทที่ 2 แรงโน้มถ่วง...") การออกแบบ Threshold จึงแก้เกมเรื่องรอยต่อของ UX ให้สมบูรณ์แบบ

## 4. Deduplication (การกำจัดข้อมูลซ้ำซ้อน)

หนังสือในโลกความเป็นจริงอาจมีการพิมพ์ทับซ้อนกัน ใน `rag_engine.py` จึงมีฟังก์ชัน `_deduplicate_passages()` 
มันทำเกณฑ์เช็คความเหมือนโดยใช้โครงสร้างข้อมูลแบบ Set: ตรวจอักษร 50 ตัวแรก หากซ้ำกับเป้าหมายที่เคยเจอแล้ว (O(1) lookup time) ระบบดรอปทิ้งทันที ช่วยปกป้องให้ AI ได้รับชุดข้อมูลที่ดีและหลากหลายที่สุด
